# TDD(Test Driven Development)
테스트 주도 개발
1. 실패하는 테스트 코드를 먼저 작성한다. 
2. 테스트 코드를 성공시키기 위한 실제 코드를 작성한다. 
3. 중복 코드 제거, 일반화 등의 리팩토링을 수행한다.
> 실제 코드에 대해 기대되는 바를 보다 명확하게 정의함으로써 불필요한 설계를 피할 수 있고, 정확한 요구 사항에 집중할 수 있다.
